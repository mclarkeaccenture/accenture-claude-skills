ℹ️ How to set up the two Power Automate flows (10 min, no IT ticket) →
Power Automate setup — bypasses DLP by using a generic HTTP action instead of the blocked Smartsheet connector
Both flows call api.smartsheet.com directly with your API token. The DLP policy blocks the Smartsheet connector — not outbound HTTP requests.
FLOW 1 — Write (HTML → Smartsheet)
1
New flow → Instant cloud flow → trigger: When an HTTP request is received. Method: POST.
2
Add action: HTTP (Premium connector). Set:
Method: POST URI: https://api.smartsheet.com/2.0/sheets/3347520112512900/rows Headers: Authorization → Bearer HjWP8vWXFHrRAyQNjB1opaKX9gJNwE3kksEKG Content-Type → application/json Body: [{"cells": @{triggerBody()?['cells']}, "toBottom": true}]
3
Add action: Response. Status code: 200. Body: {"result":"ok"}. Save — copy the HTTP POST URL → paste as Write webhook.
FLOW 2 — Read (Smartsheet → HTML)
1
New flow → Instant cloud flow → trigger: When an HTTP request is received. Method: GET.
2
Add HTTP action:
Method: GET URI: https://api.smartsheet.com/2.0/sheets/3347520112512900 Headers: Authorization → Bearer HjWP8vWXFHrRAyQNjB1opaKX9gJNwE3kksEKG
3
Add Response action. Status: 200. Body: @{body('HTTP')}. Add header: Content-Type → application/json. Save — paste URL as Read webhook.
